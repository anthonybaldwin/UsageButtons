What it does

Each metric becomes its own Stream Deck key. The background fills (or drains) in proportion to the current value, so you can read the whole dashboard without squinting.

Live dynamic icons

Every key renders as a compact SVG whose fill is driven by the current value — watch your session window drain in real time.

Smart rate-limit aware

Shared per-provider snapshot cache coalesces all button requests into one HTTP call, so adding six Claude keys still polls Anthropic once.

Watermark brand glyphs

Each button carries its provider's logo as a watermark so you can tell a Claude tile from a Codex tile at a glance, even when the meter is full.

Reset countdowns

Subvalue line shows exactly when each window resets — 2h 15m, 5d, 47s. Format adapts to the remaining duration.

Threshold alarms

Money metrics colorize automatically — orange when your balance dips below 20% of the cap, red when spending hits the monthly limit.

Standalone native binary

Compiled via bun build --compile into a single .exe / mach-o — users don't install Bun or Node. Just drop and launch.

Install

Grab the latest compiled plugin from Releases and double-click. Alternatively, clone and build from source — Bun is the only dev dependency.

Clone the repo

No Node or npm needed — Bun is the runtime and the compiler.

git clone https://github.com/anthonybaldwin/usage-buttons.git
cd usage-buttons

Install dev deps & build

Cross-compiles to a standalone .exe on Windows or native arm64 + x64 binaries on macOS.

bun install
bun run build

Link into Stream Deck

Creates a junction (Windows) or symlink (macOS) so edits to the source propagate without reinstalling.

bun run install:dev --restart

Add a button

In Stream Deck, drag the Usage Stat action onto any key, then pick a provider + metric from the Property Inspector. Configure OAuth or paste a session cookie for claude.ai extras.
