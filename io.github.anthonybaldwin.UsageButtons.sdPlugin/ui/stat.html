Authentication

Session / Weekly / Sonnet / Opus metrics come from ~/.claude/.credentials.json via api.anthropic.com. Extras (balance, overage) come from claude.ai via the Usage Buttons Helper extension. Each metric uses the source that can actually supply it — no toggle needed.

Cursor usage comes from your cursor.com session via the Usage Buttons Helper extension. Install it from the Plugin settings tab → Browser extension.

Ollama cloud usage comes from your ollama.com session via the Usage Buttons Helper extension. Install it from the Plugin settings tab → Browser extension.

Copilot reads your GitHub token automatically from ~/.config/github-copilot/hosts.json (written by gh auth login). You can also set the GITHUB_TOKEN environment variable. No manual configuration needed if gh CLI is installed and authenticated.

Codex reads your OAuth token automatically from ~/.codex/auth.json (written by codex auth login). No manual configuration. Cost-today / cost-30d metrics are estimated locally from ~/.codex/sessions/**/*.jsonl — same pattern Claude uses.

Set the OPENROUTER_API_KEY environment variable with your API key from openrouter.ai/settings/keys.

Set the WARP_API_KEY environment variable. See Warp docs.

Set the ZAI_API_TOKEN environment variable with your API token from the z.ai dashboard.

Set the KIMI_K2_API_KEY (or KIMI_API_KEY) environment variable.

Text

Replaces the subtext under the big number (e.g. "Remaining", "Used", "Prepaid"). Reset countdowns always win — if the metric has a live timer, uncheck Show reset countdown below to let the override through.

When checked, percent metrics with raw count data show the underlying count instead of "Remaining" / "Used". Auto-enables for credit-based providers (Warp, Kimi K2, z.ai, Copilot) when the API returns raw counts. Uncheck to force off.

Warn / critical thresholds

When the metric's dollar value crosses these thresholds, the tile switches to the warn / critical color. Units are raw dollars. Leave blank for defaults ($10 warn / $0 critical for balances; 80% / 100% of cap for spend meters).

Refresh

Each button can override this from the "This button" tab.

The plugin checks GitHub Releases every 6 hours for a newer version. When an update is found, all buttons show an "UPDATE" notice until you install the new version. Enable this to suppress the check entirely.

Display

Applies to every button. CodexBar's menu bar shows "N% used" but the plugin default is "N% remaining" (classic fuel-gauge style). Flip this to match your mental model — it's plugin-wide because having half your buttons read one way and half the other is confusing.

Default text sizes

Any button in the "This button" tab can override these. Blank / "Use default" on the button sticks with the plugin-wide value.

Clears valueSize / subvalueSize from every visible button so the plugin-wide defaults above take over everywhere. Buttons on inactive profiles pick up the reset the next time they appear.

Browser extension

The Usage Buttons Helper Chrome extension reads usage data from your logged-in sessions on claude.ai, cursor.com, and ollama.com. Your cookies never leave the browser — the extension proxies requests using credentials: "include", and the plugin only sees API response bodies.

Checking extension status…

Setup: Open chrome://extensions, enable Developer mode, click Load unpacked, and pick the chrome-extension/ folder inside this repo. The plugin auto-registers the native-messaging bridge on launch — no ID paste, no admin prompts.

Register writes the native-messaging manifest so browsers can talk to the plugin. Unregister cleanly removes every registry key and manifest file. Tip: unregister before removing the plugin from Stream Deck.
